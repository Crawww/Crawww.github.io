Hexo + github | Modification on Nlvi theme

花了一天时间搭成这个blog，70%都是在挑选和更改主题，毕竟好看才有写的欲望啊哈。没学过多少前端被那些layout和style整得略晕，不过一通乱来也算是摸清了些门道。第一篇就记录下过程叭。

About GitHub Pages

官方是这么说的：

"GitHub Pages is a static site hosting service that takes HTML, CSS, and JavaScript files straight from a repository on GitHub, optionally runs the files through a build process, and publishes a website."

虽然是静态站点，虽然也有些限制比如：

  • 源仓库最多1G，不然据说会受到来自Github的邮件让你reduce （有点少，基本不用考虑上传太多图片视频了）
  • 每小时只能构建10次 （本地可以调试所以还好）

但是作为一个用来学习记录的blog大致目前也够了。

具体看这个 >> https://help.github.com/cn/github/working-with-github-pages/about-github-pages

About Hexo & Nlvi

Hexo是轻量的blog framework, 安装很方便，生成静态页面也很快，总之就很干净。安装还是看别人的经验叭官方的文档对我这样的不太友好。官方 -> https://hexo.io/docs/

因为一直是用着Markdown (Typora)做笔记，Hexo个人觉得还是比较方便的，部署到Github上就是几条命令的事，安装也很快，重点是太对我Minimalism的口味。看了一些主题以及别人的blog，大致筛出来Apollo, Next.Mist, Nlvi这几个试着改了一下，还有个改编版的Apollo叫Still也还不错。Apollo真的不太好改而且简洁过头就放弃了，主要还是太爱Nlvi了。

  1. Git clone主题的repository到 ~/Hexo/themes下，可以看到hexo-theme-Nlvi的文件夹

  2. 更改~/Hexo/_config.yml下，并且修改title, subtitle, author

  3. 重建并预览

Nlvi 的调整

  1. Nlvi下其实有两个scheme，banderole和balance, 可在~/Hexo/themes/hexo-theme-Nlvi/下的_config.yml中修改，该文件会覆盖Hexo下config.yml中的设置。也可以看到很多别的custom setting

  2. 如果对默认的style颜色不满意，可以在/Hexo/themes/hexo-theme-Nlvi/source/style/_common下的variable.styl修改各个style的颜色（前提是你要在config.yml中选择它），里面还有各种背景颜色和字体的设置

  3. 更改脚注

    • 图标选择及链接

      在主题的config.yml中删减不想要的sub，或填入ID，注意是ID；或者在Hexo/themes/hexo-theme-Nlvi/layout/_partial/_footer下的contact.swig修改

    • 版权和作者

      在Hexo/themes/hexo-theme-Nlvi/layout/_partial/_footer下的copyright.swig修改

  4. 更改About可在/Hexo/public/archives中的index.html复制到Hexo/themes/hexo-theme-Nlvi/source的About中，没有需要自己新建一个About在该目录下，用editor打开该html文件，删改页面的主内容及title。

  5. 更改Post的格式可在/Hexo/themes/hexo-theme-Nlvi/source/style/_partial/_common/下的post.styl更改，减少有序和无序列表的缩进可更改ul, ol为如下
